# 06 — Matching d'équilibre

> **Le matching JOB'S ETHIC n'est pas un matching de CV. C'est un matching d'équilibre entre deux façons de travailler.**

## 1. Le changement de paradigme

### Ancien modèle (le marché actuel)
**Question posée :** *"Est-ce que ce candidat correspond au poste ?"*

Logique :
- Le recruteur définit
- Le candidat s'adapte
- Le CV est filtre principal
- Mots-clés + tri

Résultat : 1/3 des collaborations échouent. Le candidat se révèle "différent en réalité". L'environnement se révèle "différent du pitch".

### Notre modèle
**Question posée :** *"Est-ce que ces deux personnes peuvent réellement travailler ensemble de manière durable et saine ?"*

Logique :
- **Les deux côtés** sont profilés à la même profondeur
- **Aucun ne s'adapte** au sens unique du terme
- Le **comportement** prime sur le déclaratif
- Le **score** mesure une **probabilité de réussite**, pas un % de cases cochées

Résultat attendu : nos collaborations durent **2x plus longtemps** que la moyenne du marché. **C'est notre KPI principal.**

## 2. Les 3 dimensions du matching

```
                MATCHING D'ÉQUILIBRE
                       │
        ┌──────────────┼──────────────┐
        │              │              │
   ┌────▼─────┐  ┌────▼─────┐  ┌────▼──────┐
   │ EXIGENCES│  │ ATTENTES │  │COMPATIBILITÉ│
   │   du     │  │    du    │  │  HUMAINE   │
   │RECRUTEUR │  │ CANDIDAT │  │  (les 2)   │
   └──────────┘  └──────────┘  └────────────┘
```

### Dimension 1 — Les exigences réelles du recruteur

**Pas une fiche de poste embellie.** Une déclaration honnête de la réalité du terrain.

| Item | Capture |
|---|---|
| Compétences requises | Liste précise, par niveau de criticité |
| Hard skills | Outils, langages, méthodes (pas juste "Maîtrise Excel") |
| Soft skills attendus | Autonomie, communication, gestion stress (chiffrés) |
| Niveau d'autonomie | Encadré quotidien / hebdo / total |
| Pression réelle | 0-10, illustrée par questions concrètes |
| Rythme | Standard / urgent récurrent / pics intenses |
| Outils utilisés | Stack tech exact, méthodologies |
| Valeurs | 5 valeurs perçues vraies (pas la page "About" du site) |
| Budget | Fourchette, transparente |
| Style management | Directif / participatif / autonomie totale / variable |
| Environnement | Startup / scale-up / corporate / hybride |
| Turnover récent | Oui / non / partiellement transparent |

### Dimension 2 — Le fonctionnement réel du candidat

**Pas un CV.** Le profil profond construit sur 6 semaines.

| Item | Capture (issu du Voyage) |
|---|---|
| Rythme préféré | Bleu 🔵 |
| Style communication | Vert 🟢 |
| Autonomie | Jaune 🟡 |
| Limites | Violet 🟣 (le plus important) |
| Environnement préféré | Orange 🟠 |
| Tolérance pression | Rouge 🔴 |
| Style décision | Issu Acte 2 |
| Valeurs perçues | Issu Acte 2 |
| Style conflit | Issu Acte 2 |
| Posture apprentissage | Issu Acte 2 |
| Historique mission (si applicable) | Issu plateforme |
| Évolutions récentes | Issu Acte 3 (Échos) |

### Dimension 3 — La compatibilité humaine (l'invisible)

**Là où on bat tout le marché.** Pas une simple addition des deux profils — une **mesure de leur compatibilité**.

| Item | Calcul |
|---|---|
| **Compatibilité de rythme** | Le candidat marathon est mis avec un environnement sprint → ⚠️ |
| **Compatibilité communication** | Direct + Diplomate fonctionne parfois bien (complémentarité), parfois mal (incompréhensions) → contexte |
| **Compatibilité autonomie** | Candidat très autonome + management directif = échec quasi-garanti |
| **Compatibilité limites** | Si les limites du candidat sont incompatibles avec l'environnement, **disqualification douce** (pas affiché) |
| **Compatibilité valeurs** | Top 3 valeurs candidat × valeurs entreprise → score |
| **Compatibilité posture décision** | Décideur seul vs équipe collaborative |
| **Compatibilité style conflit** | Évitement vs confrontation directe |

## 3. Le score d'équilibre — formule transparente

### La composition (100 points)

| Composant | Poids | Source |
|---|---|---|
| **Compétences techniques** | 25 | CV + tests + portfolio + missions précédentes |
| **Compatibilité comportementale** | 30 | Profil profond × profil recruteur |
| **Compatibilité environnementale** | 20 | Limites candidat × environnement réel recruteur |
| **Compatibilité humaine** | 15 | Communication, autonomie, style |
| **Disponibilité & budget** | 10 | Logistique (timing, prix, géo) |

### La règle d'or

> **Un bon match = 70-85. On ne cherche pas 100.**

Pourquoi pas 100 ?
- Un score 100 est statistiquement suspect (sur-fitting)
- Un peu de friction productive est bonne pour une collaboration
- Un score 70-85 avec **zones de tension claires** est plus durable qu'un 95 fragile

## 4. L'audit honnêteté employeur — notre arme secrète

### Le problème
Les fiches de poste mentent.
- "Environnement dynamique" = chaos quotidien
- "Autonomie" = "on ne sait pas comment vous manager"
- "Esprit d'équipe" = "tout le monde fait tout, mal"
- "Salaire compétitif" = "moyen pour le marché, bas pour le poste"

### Notre approche : forcer la vérité

Le recruteur passe par un **profilage employeur** de 15-20 min qui pose des questions très concrètes :

#### Sur la pression
- *"Sur 10, quel est le niveau de stress moyen perçu dans l'équipe en ce moment ?"*
- *"Combien d'heures supp ont été faites le mois dernier dans cette équipe ?"*
- *"Quand est-ce que vous avez perdu un collaborateur sur un sujet de surcharge ces 12 mois ?"*

#### Sur le management
- *"Vous donnez un feedback formel à votre équipe à quelle fréquence ?"*
- *"Vous tolérez quelle proportion d'erreurs (sur un livrable typique) ?"*
- *"Vous prenez vos décisions en consultant ou en autonomie ?"*

#### Sur l'environnement
- *"Décrivez votre semaine type, pas idéale."*
- *"Quel est le turnover ces 12 derniers mois ?"*
- *"Y a-t-il des tensions actuellement non résolues dans l'équipe ?"* (oui/non avec option de contexte)

### Le "Score de transparence employeur"
Le recruteur a un score visible :
- **Transparence élevée** (verts vifs) — il dit la vérité, ses matchs précédents l'ont confirmé
- **Transparence standard** (en construction) — premier usage, on n'a pas encore de signal
- **Transparence variable** (orange) — des écarts ont été remontés par les freelances précédents
- **Transparence basse** (rouge) — décalage récurrent entre déclarations et expérience freelances

**Conséquences :**
- Un recruteur avec transparence rouge perd sa **priorité de matching**
- Les freelances voient le score quand ils consultent le profil
- Le recruteur peut **améliorer son score** en ajustant ses déclarations

### Mécanique de vérification croisée
Après chaque mission terminée, le freelance répond à 3 questions :
1. La pression était-elle conforme à ce qui était annoncé ? (Beaucoup moins / Moins / Conforme / Plus / Beaucoup plus)
2. Le style de management était-il celui annoncé ? (Oui / partiellement / différent)
3. L'environnement était-il celui décrit ? (Oui / partiellement / différent)

**Ces feedbacks alimentent le score de transparence en privé.** Ils ne sont pas publics — pas de "notation toxique".

## 5. Le moteur de matching technique

### Architecture

```
┌─────────────────────────────────────────────────┐
│              MOTEUR DE MATCHING                  │
├─────────────────────────────────────────────────┤
│                                                  │
│  1. CANDIDATS DISPONIBLES                       │
│     Filtrage hard : skills, disponibilité,      │
│     limites incompatibles → exclusion           │
│                                                  │
│  2. SCORE DE COMPATIBILITÉ                      │
│     XGBoost trained sur historique missions     │
│     → score sur 100 par dimension               │
│                                                  │
│  3. EXPLICATION (LLM)                           │
│     Claude Opus 4.7 prend score + features      │
│     → produit explication langage naturel       │
│                                                  │
│  4. TOP-K SÉLECTION                             │
│     Garde top 3 (jamais plus)                   │
│     Diversifie si scores similaires             │
│                                                  │
│  5. PROJECTION DE COLLABORATION (V2)            │
│     Modèle prédictif durée + risques            │
│                                                  │
└─────────────────────────────────────────────────┘
```

### Stack technique recommandé

| Composant | Technologie | Pourquoi |
|---|---|---|
| **Algorithme principal** | XGBoost / LightGBM | Auditable, explicable via SHAP, robuste sur petites données |
| **Features d'entrée** | Vecteurs profil + vecteurs recruteur (~80 features) | Mix continu + catégoriel |
| **Embeddings textuels** | Embeddings OpenAI (text-embedding-3) | Pour textes libres (fiche de poste, profil texte) |
| **Cohérence longitudinale** | Réseau de neurones simple PyTorch | Apprend les patterns de cohérence sur 30+ sessions |
| **Explication LLM** | Claude Opus 4.7 | Prend SHAP values + contexte, produit explication |
| **Réentraînement** | Hebdo au début, mensuel à scale | Sur les résultats de mission (durée + satisfaction) |

### Données d'entraînement initiales
**Problème classique :** au démarrage, on n'a **pas** de données de mission.

**Solution :**
- **Phase 0-1 (matchs à la main)** : tu génères tes premières 50 missions toi-même. Chaque mission produit 1 ligne de "vrai match + résultat".
- **Phase 2 (V1 lancé)** : on simule + scrape de la littérature psychométrique pour bootstrap, mais on **utilise toujours** l'humain dans la boucle pendant 3 mois.
- **Phase 3 (>200 missions terminées)** : l'algorithme prend la main, supervisé par toi.

### Anti-biais structurel
- **Audit régulier** des features qui contribuent le plus → si une corrèle indirectement avec un critère sensible (ex : ville, langue maternelle, photo), on la **retire ou pondère**
- **Test fairness** trimestriel : différentiel de scoring entre groupes démographiques
- **Pas de réseaux profonds opaques** sur la décision finale — XGBoost est explicable via SHAP

## 6. Présentation du matching — UX clé

### La règle "3 profils, 1 choix, 10 minutes"

Quand un recruteur poste une mission, il reçoit **3 profils** (jamais plus, jamais moins).

**Pourquoi 3 ?**
- Plus que 1 : laisse un choix réel
- Moins que 5 : pas de surcharge cognitive
- Le marché actuel donne 30+ profils sans hiérarchie → on inverse

**Si moins de 3 profils sont disponibles :**
- On dit la vérité : *"Seulement 2 profils correspondent réellement à votre besoin. Mieux 2 vrais matchs que 5 forcés."*
- Option : étendre les critères (alerte au recruteur)
- Option : alerter quand un nouveau profil compatible s'inscrit

**Si plus de 3 profils sont disponibles :**
- On garde le top 3 par score
- On peut afficher *"5 autres profils sont à proximité"* — accessible sur clic, jamais en surcharge

### La page de comparaison
Le recruteur peut comparer les 3 profils côte à côte, avec :
- Score global + scores par dimension
- 3 forces de chacun
- 2 zones de tension de chacun
- Recommandation contextuelle

**Volontairement minimaliste.** Pas de tableau Excel-like.

## 7. La simulation de collaboration (V2)

### Principe
Avant que la collaboration commence, l'IA produit une **projection** :

```
┌────────────────────────────────────────────────┐
│  SIMULATION DE VOTRE COLLABORATION             │
│  Toi (recruteur) × Émilie (freelance)          │
├────────────────────────────────────────────────┤
│                                                │
│  Durée estimée de la mission        ~3 mois    │
│  Probabilité de seconde mission       72 %     │
│  Risque de rupture anticipée          18 %     │
│                                                │
│  ─── POINTS FORTS ATTENDUS ───                 │
│  ◆ Réactivité — vos rythmes alignent           │
│  ◆ Communication écrite Slack — préférée 2x    │
│  ◆ Style décision — autonomie respectée        │
│                                                │
│  ─── RISQUES IDENTIFIÉS ───                   │
│  ◆ Pression : son seuil est 4/10, le tien 7    │
│    → Faire un point à J+15 pour ajuster        │
│  ◆ Feedback : elle préfère hebdo,              │
│    ton équipe fait à la demande                │
│    → Mettre un Slack reminder hebdo            │
│                                                │
│  ─── CONDITIONS DE RÉUSSITE ───                │
│  1. Cadrage clair du périmètre Semaine 1       │
│  2. Point hebdo court (15 min) le mardi        │
│  3. Pas de demande urgente le week-end         │
│                                                │
└────────────────────────────────────────────────┘
```

### Construction
- **Données d'entrée** : score + features du match + historique de matchs similaires
- **Modèle** : régression sur les missions précédentes "match similaire" → durée + résultat
- **Format** : LLM produit le rendu en langage naturel à partir des prédictions numériques

### Pourquoi c'est WAOUH
- Personne ne fait ça aujourd'hui (Pymetrics, AssessFirst, etc.)
- C'est **actionnable** (pas juste un score) — le recruteur sait quoi faire
- C'est **partageable** entre recruteur et freelance avant signature → alignement
- Ça crée de la **confiance** dans la décision

## 8. La contestabilité — pilier 4 opérationnel

### Droit de contester
**Tout score peut être contesté** par les deux parties dans les 30 jours :
- Par formulaire structuré
- Avec motif (pas obligatoire mais recommandé)
- Sans représailles ni baisse de visibilité

### Processus
1. Le candidat (ou recruteur) clique "Contester le score"
2. Formulaire : "Quelle dimension semble incorrecte ? Pourquoi ?"
3. Sous 48h : un humain de l'équipe JOB'S ETHIC examine
4. Si jugé légitime :
   - Le score est ajusté
   - Le profil profond est mis à jour si nécessaire
   - La donnée alimente un audit pour améliorer le modèle
5. Si jugé non légitime :
   - Explication détaillée envoyée
   - Le score reste

### Pourquoi c'est crucial
- **Conformité AI Act native** (right to explanation + right to contest)
- **Conformité GDPR Art. 22**
- **Brand** : "vous pouvez toujours contester un score"
- **Amélioration** : chaque contestation est un signal d'apprentissage

## 9. Les KPIs du matching

### KPIs produit (mesurés en continu)

| KPI | Cible 12 mois |
|---|---|
| Score moyen des matchs proposés | ≥ 75 |
| Taux de matchs menant à un contact | ≥ 60 % |
| Taux de matchs menant à une mission | ≥ 35 % |
| Durée moyenne mission JOB'S ETHIC | ≥ 2× marché |
| % missions menant à 2ème mission | ≥ 30 % |
| NPS recruteur | ≥ 50 |
| NPS freelance | ≥ 60 |
| Taux de contestation de score | < 5 % |

### KPIs anti-biais (mesurés trimestriellement)

| KPI | Tolérance |
|---|---|
| Écart de score moyen H/F sur mêmes profils | < 2 points |
| Écart de score moyen par origine perçue | < 3 points |
| Écart de score moyen entre français natif / non-natif | < 4 points |
| Taux de matchs aboutis par démographie | différentiel < 10 % |

**Tout dépassement déclenche un audit.**
